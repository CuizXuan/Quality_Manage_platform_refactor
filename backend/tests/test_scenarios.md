# 场景管理模块测试用例

## 功能描述
场景管理模块负责将多个用例编排成场景，支持步骤管理、变量提取、失败跳过、重试机制，可批量执行。

## API 端点
| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/scenarios` | 获取场景列表 |
| POST | `/api/scenarios` | 创建场景 |
| GET | `/api/scenarios/{scenario_id}` | 获取场景详情（含步骤） |
| PUT | `/api/scenarios/{scenario_id}` | 更新场景 |
| DELETE | `/api/scenarios/{scenario_id}` | 删除场景 |
| POST | `/api/scenarios/{scenario_id}/steps` | 添加步骤 |
| PUT | `/api/scenarios/{scenario_id}/steps/{step_id}` | 更新步骤 |
| DELETE | `/api/scenarios/{scenario_id}/steps/{step_id}` | 删除步骤 |
| PUT | `/api/scenarios/{scenario_id}/steps/reorder` | 步骤排序 |
| POST | `/api/scenarios/{scenario_id}/run` | 执行场景 |

## 边界条件清单
- 场景无步骤时执行
- 步骤引用不存在的用例
- 循环引用场景（如果支持）
- 步骤 order 超序
- 变量提取规则格式错误
- retry_times=0 或负数
- retry_interval=0 或负数

## 异常场景清单
- 场景不存在（404）
- 用例不存在（404）
- 环境不存在（使用默认环境或空）
- 执行超时
- 目标服务不可达
