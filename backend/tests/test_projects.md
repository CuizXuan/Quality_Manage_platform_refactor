# 项目管理模块测试用例

## 功能描述
项目管理模块提供项目的创建、查询、更新、删除及成员管理功能，支持分页、关键词过滤、状态过滤。

## API 端点
| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/projects` | 获取项目列表（分页、过滤） |
| POST | `/api/projects` | 创建项目 |
| GET | `/api/projects/{project_id}` | 获取单个项目详情 |
| PUT | `/api/projects/{project_id}` | 更新项目 |
| DELETE | `/api/projects/{project_id}` | 删除项目（软删除） |
| GET | `/api/projects/{project_id}/members` | 获取项目成员列表 |
| POST | `/api/projects/{project_id}/members` | 添加项目成员 |
| PUT | `/api/projects/{project_id}/members/{user_id}` | 更新成员角色 |
| DELETE | `/api/projects/{project_id}/members/{user_id}` | 移除项目成员 |

## 边界条件清单
- 分页 page=0 或 page=-1
- page_size=0、page_size=101（超过最大100）
- key 超长（>50字符）
- name 超长（>100字符）
- 关键词超长
- status 值为非法枚举

## 异常场景清单
- 未登录访问（401）
- Token 过期（401）
- 无权限操作（403）
- 资源不存在（404）
- 重复资源创建（400）
- 参数校验失败（422）

## 测试数据准备
- 至少2个租户（Tenant A, Tenant B）
- 每个租户至少2个用户（Admin, Member）
- 每个租户至少3个项目（不同状态：active, archived）
- 每个项目至少2个成员
