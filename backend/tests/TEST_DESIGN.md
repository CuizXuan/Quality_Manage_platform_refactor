## 1. 测试范围

| 模块 | 描述 | API 前缀 |
|------|------|----------|
| 项目管理 | 项目的增删改查、成员管理 | `/api/projects` |

| 定时任务 | 定时调度、立即执行、启停控制 | `/api/schedules` |

## 2. 测试用例设计（按模块）

### A. 项目管理模块

#### 功能描述
项目管理模块提供项目的创建、查询、更新、删除及成员管理功能，支持分页、关键词过滤、状态过滤。

#### API 端点
| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/projects` | 获取项目列表（分页、过滤） |
| POST | `/api/projects` | 创建项目 |
| GET | `/api/projects/{project_id}` | 获取单个项目详情 |
| PUT | `/api/projects/{project_id}` | 更新项目 |
| DELETE | `/api/projects/{project_id}` | 删除项目（软删除） |
| GET | `/api/projects/{project_id}/members` | 获取项目成员列表 |
| POST | `/api/projects/{project_id}/members` | 添加项目成员 |
| PUT | `/api/projects/{project_id}/members/{user_id}` | 更新成员角色 |
| DELETE | `/api/projects/{project_id}/members/{user_id}` | 移除项目成员 |

#### 边界条件
- 分页 page=0 或 page=-1
- page_size=0、page_size=101（超过最大100）
- key 超长（>50字符）
- name 超长（>100字符）

#### 异常场景
- 未登录访问（401）
- Token 过期（401）
- 无权限操作（403）
- 资源不存在（404）
- 重复资源创建（400）
- 参数校验失败（422）

### B. 用例管理模块

#### 功能描述
用例管理模块负责测试用例的 CRUD 操作，包括用例的创建、执行、复制、批量删除等功能。用例与项目通过 folder_path 关联。

| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/cases` | 获取用例列表（过滤、分页） |

| POST | `/api/cases/batch-delete` | 批量删除用例 |
| POST | `/api/cases/{case_id}/run` | 执行用例 |

#### 边界条件
- limit=0、limit=501（超过最大500）
- skip 超出现有数据量
- headers/params/body 为空对象 `{}`
- assertions 为空数组 `[]`
- method 为空或不合法
- url 超长

#### 异常场景
- URL 格式不合法
- 超时设置异常（timeout=0 或负数）
- 认证配置格式错误
- 执行时目标服务不可达（应记录失败状态）

### C. 缺陷管理模块

#### 功能描述
缺陷管理模块负责缺陷的创建、跟踪、状态流转，支持从执行记录一键创建缺陷，包含评论和附件功能。

| 方法 | 端点 | 功能 |

| GET | `/api/defects` | 获取缺陷列表（过滤） |

| DELETE | `/api/defects/{defect_id}` | 删除缺陷 |
| POST | `/api/defects/{defect_id}/comments` | 添加评论 |
| POST | `/api/defects/{defect_id}/attachments` | 上传附件 |
| POST | `/api/defects/from-execution` | 从执行记录创建缺陷 |
| GET | `/api/defects/stats/summary` | 缺陷统计看板 |

#### 边界条件
- cron 表达式格式错误
- target_id 指向不存在资源
- 任务列表分页 skip 超界
- limit=0 或 limit>500

#### 异常场景
- 任务不存在（404）
- target 不存在（404）
- 执行失败（应记录 failure_count）
- 无权限操作（403）

## 3. 测试执行优先级

### P0 - 核心流程（阻断性测试）
| 模块 | 测试点 |
|------|--------|
| 项目管理 | 创建项目、获取项目列表、获取单个项目、更新项目、删除项目 |
| 用例管理 | 创建用例、获取用例列表、获取单个用例、更新用例、删除用例、执行用例 |
| 缺陷管理 | 创建缺陷、获取缺陷、更新缺陷状态（resolved）、删除缺陷 |
| 场景管理 | 创建场景、获取场景、执行场景 |
| 环境管理 | 创建环境、获取环境列表、获取单个环境、删除环境、设置默认环境 |
| 定时任务 | 创建任务（用例/场景）、获取任务列表、立即执行 |

### P1 - 重要功能（正常流程测试）
- 各模块的过滤、分页、排序功能
- 关联关系测试（成员管理、步骤管理）
- 状态流转测试
- 统计功能测试

### P2 - 边界情况（异常测试）
- 参数边界值测试
- 权限控制测试
- 资源不存在测试
- 数据关联测试

### P3 - 低优先级（优化测试）
- 性能相关（大数据量）
- UI 相关（前端交互）
- 日志审计
