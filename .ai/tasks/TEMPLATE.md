# Codex 实现任务包

## 任务目标

描述 Claude 需要实现的功能、修复的缺陷或执行的重构。

## 背景说明

说明用户侧问题、业务背景和相关产品行为。

## 必读文件

- `AGENTS.md`
- 在这里列出相关后端文件。
- 在这里列出相关前端文件。
- 在这里列出相关测试文件。

## 允许修改范围

- 列出 Claude 可以修改的文件、目录或模块。

## 禁止事项

- 不做无关重构。
- 除非明确要求，不改变公开 API 响应结构。
- 除非明确要求，不引入新依赖。
- 未先说明原因前，不修改允许范围之外的文件。

## 实现要求

- 函数保持短小，职责单一。
- 路由函数保持轻薄。
- 业务逻辑放在 service 层。
- 前端 API 调用放在 `frontend/src/api/`。
- Vue 组件保持职责聚焦。
- 对用户输入进行校验。
- 保持任务范围外的现有行为不变。
- 对用户可见的说明、计划、总结、测试结果必须使用中文。

## 验收标准

- 在这里写清楚可判断通过或失败的标准。
- 覆盖成功路径。
- 覆盖异常路径。
- 如果涉及前端，写清楚前端交互和展示行为。

## 验证方式

运行最相关的命令，例如：

```text
cd backend && pytest
cd frontend && npm run build
```

如果命令无法运行，说明阻塞原因和风险。

## Claude 输出要求

结束时必须用中文汇报：
- 变更文件。
- 已运行的测试或检查。
- 每项检查的结果。
- 剩余风险。
