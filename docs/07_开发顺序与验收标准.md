# 开发顺序与验收标准

## 1. 第一阶段：平台底座

状态：已初始化。

验收：

- 登录可用
- 默认管理员可登录
- 用户管理可增删改查
- 角色管理可增删改查
- 组织管理可增删改查
- 菜单管理可增删改查
- 深浅主题可切换

## 2. 第二阶段：终端调试台

开发内容：

- 请求编辑器
- 响应查看器
- 调试历史
- 收藏请求
- 保存为基础用例
- 断言建议

验收：

- 可以发送 GET/POST 请求
- 可以查看响应
- 可以保存为用例
- 保存后的用例可在用例中心查看

## 3. 第三阶段：用例中心

开发内容：

- 用例 CRUD
- 文件夹
- 标签
- 断言
- 变体
- 执行

验收：

- 基础用例可编辑
- 可派生边界值和异常值变体
- 用例和变体可执行
- 执行结果进入历史

## 4. 第四阶段：场景编排

开发内容：

- 场景 CRUD
- 步骤编排
- 变量提取
- 变量注入
- 场景执行

验收：

- 可把多个用例组成场景
- 登录 token 可传递给后续步骤
- 场景执行有完整步骤结果

## 5. 第五阶段：报告、缺陷、门禁

开发内容：

- 报告中心
- 缺陷中心
- 质量门禁

验收：

- 执行历史可生成报告
- 失败项可创建缺陷
- 门禁可根据报告和缺陷给出放行结果

## 6. 第六阶段：AI 中枢

开发内容：

- 模型配置
- Prompt 模板
- 用例生成
- 变体生成
- 断言生成
- 失败归因
- 报告总结

验收：

- AI 输出结构化 JSON
- AI 建议可预览
- AI 建议可采纳
- 采纳动作可追溯

## 7. 每个模块的完成定义

一个模块完成必须满足：

- 页面可用
- API 可用
- CRUD 完整
- 权限编码明确
- 错误提示清晰
- 关键路径有测试
- 与上下游模块至少打通一条链路
