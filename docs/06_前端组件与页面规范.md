# 前端组件与页面规范

> 这是前端页面与组件改造的总入口。  
> 详细的 vibecoding 页面模板、公共组件边界、尺寸规范和复用规则，见 [页面改造与组件复用规范_vibecoding.md](./页面改造与组件复用规范_vibecoding.md)。

## 0. 使用说明

给模型分派前端任务时，优先指定本文件作为主规范。

- 新页面改造、旧页面重构、组件拆分，先读本文件
- 如果任务涉及页面骨架、查询栏、表格、弹窗、字体、背景、公共样式，再补读 `页面改造与组件复用规范_vibecoding.md`
- 如果任务只改某个模块，同时指定模块文件和本规范文件

推荐任务写法：

- `请基于 docs/06_前端组件与页面规范.md 和 docs/页面改造与组件复用规范_vibecoding.md，改造 xxx 页面`
- `请参考 frontend/src/views/case/CaseTable.vue 的表格规范，抽成公共组件并写入对应文档`

如果只允许指定一个文件，就用 `docs/06_前端组件与页面规范.md`。

## 1. 核心原则

### 1.1 页面是工作台，不是展示页

后台业务页面必须优先服务高频操作、扫描、筛选、批量处理和编辑效率。

- 首屏必须直接展示可操作内容，不能做营销式 hero。
- 信息层级固定为：标题区 -> 指标区 -> 视图切换 -> 主工作区。
- 主工作区必须铺满剩余空间，避免页面下方出现无意义空白。
- 页面内所有面板使用紧凑、清晰、稳定的工作台密度。

### 1.2 官方组件优先

基础控件必须优先使用 Vue 官方能力和 Element Plus 官方组件。

| 场景 | 必须使用 |
| --- | --- |
| 输入框 | `el-input` |
| 下拉 | `el-select` + `el-option` |
| 日期 | `el-date-picker` |
| 树 | `el-tree` / `el-tree-select` |
| 表格 | `el-table` + `el-table-column` |
| 分页 | `el-pagination` |
| 标签页 | `el-tabs` + `el-tab-pane` |
| 弹窗 | `el-dialog` |
| 开关 | `el-switch` |
| 提示 | `ElMessage` / `ElMessageBox` |
| 图标 | Element Plus icons |

禁止手写替代基础组件。

### 1.3 组件只封装业务组合

允许封装业务组合组件，不允许封装空壳 UI 包装。

可复用组件示例：

- `CaseStatCards`
- `CaseFilterBar`
- `CaseSearchBar`
- `CaseTable`
- `OverflowText`
- `RichTextEditor`
- `PageSurface`

拆分标准：

- 一个组件只负责一个明确区域。
- 组件内部函数保持简短，事件向外抛出。
- 页面组件负责数据请求与组合，不把所有 UI 堆在一个文件里。
- 能复用到其他模块的结构，应沉淀为同风格业务组件。

## 2. 页面骨架

推荐页面骨架：

1. 标题区
2. 可选统计区
3. 可选分类区
4. 查询区
5. 列表区
6. 弹窗区

不是所有页面都要有分类树、统计区、侧栏和高级区。

### 2.1 页面容器

页面容器必须占满父容器。

```css
.page-shell {
  display: flex;
  width: 100%;
  height: 100%;
  min-height: 0;
  min-width: 0;
  flex-direction: column;
  overflow: hidden;
}
```

### 2.2 面板标准

面板统一使用玻璃质感：

- 圆角：`8px`
- 边框：`1px`
- 阴影：轻阴影 + 内发光
- 背景：深浅色自适配

推荐通过 `PageSurface` 复用。

## 3. 字体规范

- 页面正文：`14px`
- 表单标签：`14px`
- 表格内容：`14px`
- 表格表头：`13px`
- 弹窗标题：`16px`
- 区块标题：`15px`
- 辅助说明：`12px ~ 13px`

要求：

- 不要把正文做得过小。
- 同一页面不要混太多字号。
- 重点字段要比辅助字段更清晰。

## 4. 查询栏规范

### 4.1 推荐结构

- 左侧：筛选条件
- 右侧：操作按钮

### 4.2 常见尺寸

- 输入框 / 选择框：`34px ~ 36px`
- 普通单选：`220px ~ 280px`
- 关键字输入：`260px ~ 320px`
- 日期区间：`320px ~ 360px`
- 按钮：`34px`

### 4.3 交互规则

- 默认不自动请求。
- 点击“查询”后再请求。
- 回车可触发查询。
- 点击“重置”清空条件并请求。
- 分页变化可触发请求。

### 4.4 草稿条件与已应用条件

必须区分：

- `draftFilters`：用户正在编辑的条件
- `appliedFilters`：已经提交给后端的条件

构造请求时必须使用 `appliedFilters`。

## 5. 表格规范

### 5.1 推荐列宽

- 编号：`140px ~ 170px`
- 名称：`180px ~ 240px`
- 方法 / 状态：`80px ~ 120px`
- 长文本：`220px ~ 320px`
- 日期：`140px ~ 160px`
- 创建人：`90px ~ 120px`
- 操作：`120px ~ 150px`

### 5.2 规则

- 编号和名称分列展示。
- 长文本统一用 `OverflowText`。
- 操作列固定且紧凑。
- hover 高亮必须与主题一致。
- 空状态必须统一。

### 5.3 编号规则

- 接口用例：`APICASE-YYYYMMDD01`
- 功能用例：`FUNCCASE-YYYYMMDD01`

## 6. 弹窗规范

### 6.1 弹窗尺寸

- 简单表单：`760px`
- 标准表单：`960px`
- 复杂表单：`width: min(1280px, 95vw)`
- 顶部间距：`3vh ~ 6vh`
- 内容区高度：`70vh ~ 80vh`

### 6.2 字段分区

- 1 到 6 个字段：单列或双列
- 7 到 12 个字段：双列 + 分区
- 超过 12 个字段：折叠区或标签页

### 6.3 分层

- 基础信息
- 核心配置
- 高级配置

### 6.4 行为

- 打开时回填数据。
- 保存后刷新列表。
- 编辑态和新增态标题区分。
- 关闭时清理临时状态。

## 7. 背景与质感

推荐背景结构：

1. 主题底色
2. 细网格底纹
3. 轻扫描线
4. 极淡节点流线

要求：

- 背景弱于正文。
- 动效轻。
- 深浅色都可用。
- 优先 CSS / 伪元素，不依赖真实图片。

## 8. 公共组件

### 8.1 `OverflowText`

用途：

- 单行截断
- 多行换行
- 悬停显示完整文本

适用场景：

- 编号
- 名称
- URL
- 描述
- 前置条件
- 创建人

### 8.2 `RichTextEditor`

用途：

- 富文本说明
- 前置条件
- 结果描述

### 8.3 `PageSurface`

用途：

- 页面面板外壳
- 玻璃质感容器
- 科技纹理容器

## 9. 主题适配

必须验证：

- 输入框
- 下拉框
- 日期框
- tooltip
- table hover
- dialog
- 选中态

要求：

- 不写死颜色。
- 所有颜色优先走变量。
- 浮层必须兼容深浅色。
- 不允许主题切换后出现高亮污染。

## 10. Vibecoding 顺序

建议按以下顺序生成页面：

1. 页面骨架
2. 查询区
3. 列表区
4. 弹窗区
5. 背景与动效

没有的模块直接删，不要硬造。

## 11. 复用边界

### 11.1 可以直接复用

- `OverflowText`
- `RichTextEditor`
- `PageSurface`
- `page-framework.css`
- `variables.css`
- `global.css`
- `element-override.css`

### 11.2 只能作为业务参考

- `CaseSearchBar`
- `CaseFilterBar`
- `CaseTable`
- `CaseSidebar`
- `CaseEditDialog`
- `CaseStatCards`

这些组件承载了当前用例管理的业务逻辑和排布方式，后续模块可以参考结构，但不要直接照搬字段。

## 12. 验收标准

页面改造完成后，应满足：

- 字体清晰
- 布局均匀
- 查询栏整齐
- 表格可扫读
- 弹窗分区明确
- 深浅色主题一致
- tooltip 正常
- 没有样式污染
