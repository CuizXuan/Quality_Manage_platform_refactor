# 前端组件与页面规范

## 1. 技术栈

- Vue 3
- Vite
- Pinia
- Vue Router
- Element Plus
- Axios

## 2. 目录结构

```text
frontend/src/
├── api/
├── app/
├── components/
├── router/
├── stores/
├── views/
└── style.css
```

## 3. 页面设计原则

- 登录页保持现状
- 业务页使用 App Shell
- 页面是工作台，不是营销页
- 高密度但要有清晰层级
- 不使用大面积装饰图
- 不使用卡片套卡片
- 支持深色和浅色主题

## 5. Store 规范

每个领域一个 Store：

- `useAuthStore`
- `useSystemStore`
- `useTerminalStore`
- `useCaseStore`
- `useScenarioStore`
- `useEnvironmentStore`
- `useExecutionStore`
- `useAiStore`

Store 只负责状态和调用 API，不写复杂 UI 逻辑。

## 6. API 规范

每个领域一个 API 文件：

- `auth.js`
- `system.js`
- `terminal.js`
- `case.js`
- `scenario.js`
- `environment.js`
- `execution.js`
- `ai.js`

API 文件只封装请求，不处理页面状态。

## 7. 样式规范

- 统一使用 CSS 变量
- 字体不随视口缩放
- 中文使用 `PingFang SC` / `Microsoft YaHei`
- 英文数字使用 `Inter`
- 代码使用 `JetBrains Mono`
- 表格、按钮、抽屉、弹窗统一半径和间距
- 状态色统一：成功、失败、警告、进行中、禁用

## 8. 交互规范

- 所有提交动作必须有 loading
- 所有删除动作必须二次确认
- 所有空状态必须给下一步动作
- 所有执行动作必须能查看日志
- 所有 AI 建议必须先预览再采纳
