质量管理平台前端样式与布局规范
===============

1. 总体原则

-------

* **UI 组件优先**：布局、表单、表格、弹窗等一律使用 Element Plus 组件，避免用原生 div 模拟。

* **CSS 变量驱动**：所有颜色、间距、圆角、阴影等视觉属性必须通过 CSS 变量定义，禁止硬编码色值。

* **scoped 隔离**：每个 Vue 组件的样式使用 `<style scoped>`，避免样式泄露；全局样式只存放主题变量、布局结构和 Element Plus 覆盖。

* **深浅色原生支持**：通过一套 CSS 变量 + 根类切换实现，不引入额外 UI 库。

* * *

2. 目录结构（样式相关）

-------------

text

frontend/src/
├── styles/

└── style.css              # 入口文件（仅引入上述 styles/ 目录）

> 注意：删除原有的零散 `style.css`，改为模块化引入。

### 3.3 使用约束

* **任何组件（包括 Element Plus 内部）** 的样式自定义，只能通过修改上述 CSS 变量实现，禁止直接写 `color: #333` 或 `background: white`。

* 需要新增视觉属性时，先在 `variables.css` 中定义变量，再使用 `var(--xxx)` 引用。

### 4.2 左侧菜单（`MenuSidebar.vue`）

* 使用 **Element Plus 的 `el-menu`** 组件，属性：`collapse`（支持折叠）、`background-color`、`text-color`、`active-text-color`。

* 菜单数据来自后端或前端静态配置，结构为树形（`children`）。

* **多级菜单规则**：
  
  * 一级菜单分组（无 `children` 的直接作为可点击页面）。
  
  * 超过两级时，不再在侧栏嵌套缩进，而是在目标页面内使用二级 Tabs 或左侧子面板（例如用例中心采用 `el-tabs` 区分“用例列表”和“用例详情”）。

* 菜单项必须包含 `icon`（使用 Element Plus 图标库）、`index`（路由路径）、`title`（中文名称）。

vue

<template>
  <el-menu
    :default-active="$route.path"
    :collapse="isCollapse"
    :unique-opened="true"
    background-color="var(--bg-sidebar)"
    text-color="var(--text-primary)"
    active-text-color="var(--el-color-primary)"
  >
    <el-sub-menu v-for="item in menuList" :key="item.path" :index="item.path">
      <template #title>
        <el-icon><component :is="item.icon" /></el-icon>
        <span>{{ item.title }}</span>
      </template>
      <el-menu-item v-for="child in item.children" :index="child.path">
        <el-icon><component :is="child.icon" /></el-icon>
        <span>{{ child.title }}</span>
      </el-menu-item>
    </el-sub-menu>
  </el-menu>
</template>

* **高亮与选中态**：使用 `el-menu` 自带的 `active-text-color` 为 `var(--el-color-primary)`，背景色不额外改变。

5. 页面组件样式规范

-----------

### 5.1 页面布局（以终端调试台为例）

每个页面内部使用 **Element Plus 的 `el-row` / `el-col`** 或 `flex` 实现分栏，不用手写复杂 CSS。

vue

<template>
  <div class="debugger-page">
    <el-row :gutter="16">
      <el-col :span="8">
        <HistoryPanel />
      </el-col>
      <el-col :span="16">
        <RequestPanel />
        <ResponsePanel />
        <AiAnalysisPanel />
      </el-col>
    </el-row>
  </div>
</template>
<style scoped>
.debugger-page {
  height: 100%;
  background-color: var(--bg-page);
}
/* 子组件内部样式也使用 scoped，且只使用 CSS 变量 */
</style>

### 5.2 组件内部样式写法

* 类名使用 **BEM 或短横线命名**（如 `.request-panel__header`）。

* 禁止使用 `!important`。

* 所有自定义样式必须基于 CSS 变量，例如：

css

.request-panel {
  border: 1px solid var(--border-color);
  border-radius: var(--border-radius-base);
  background-color: var(--bg-container);
  padding: var(--spacing-md);
  margin-bottom: var(--spacing-md);
}

* 对于表格、表单等复杂组件，优先使用 Element Plus 提供的属性（如 `stripe`、`border`），不自己造轮子。

### 5.3 长文本、截断与省略

全局类名：

css

/* styles/global.css */
.text-ellipsis {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
  max-width: 100%;
}
.cell-ellipsis {
  max-width: 200px; /* 根据需要覆盖 */
}

在表格列中：

vue

<el-table-column prop="description" label="描述">
  <template #default="{ row }">
    <span :title="row.description" class="text-ellipsis">{{ row.description }}</span>
  </template>
</el-table-column>

6. 深浅色主题切换实现

------------

### 6.1 切换逻辑（`composables/useTheme.ts`）

typescript

import { ref, watch } from 'vue';
export const useTheme = () => {
  const isDark = ref(localStorage.getItem('theme') === 'dark');
  const setTheme = (dark: boolean) => {
    const html = document.documentElement;
    if (dark) {
      html.classList.add('dark');
    } else {
      html.classList.remove('dark');
    }
    localStorage.setItem('theme', dark ? 'dark' : 'light');
    isDark.value = dark;
  };
  // 监听系统配色（可选）
  const matchMedia = window.matchMedia('(prefers-color-scheme: dark)');
  matchMedia.addEventListener('change', (e) => {
    if (!localStorage.getItem('theme')) {
      setTheme(e.matches);
    }
  });
  return { isDark, setTheme, toggle: () => setTheme(!isDark.value) };
};

### 6.2 在顶部栏添加切换按钮

vue

<el-button text :icon="isDark ? Sunny : Moon" @click="toggleTheme" />

### 6.3 注意 Element Plus 暗色适配

Element Plus 内置暗色主题变量，但我们已通过 CSS 变量覆盖保持一致。确保在 `main.ts` 中引入：

js

import 'element-plus/theme-chalk/dark/css-vars.css';

这样 Element Plus 内部组件也会自动跟随 `.dark` 类变化。

7. 菜单栏排列详细规范

------------

### 7.1 菜单结构示例（静态配置或后端返回）

js

// stores/menu.js
export const menuList = [
  {
    title: '工作台',
    icon: 'Monitor',
    path: '/dashboard',
    children: []  // 无子菜单时为普通菜单项
  },
  {
    title: '工具',
    icon: 'Tools',
    children: [
      { title: '终端调试台', icon: 'Connection', path: '/terminal' },
      { title: '用例中心', icon: 'Document', path: '/case' }
    ]
  },
  {
    title: '系统管理',
    icon: 'Setting',
    children: [
      { title: '用户管理', path: '/system/user' },
      { title: '角色管理', path: '/system/role' },
      { title: '菜单管理', path: '/system/menu' }
    ]
  }
];

### 7.2 菜单权限控制

* 菜单渲染时根据后端返回的权限列表过滤（在 store 中维护 `accessibleMenus`）。

* 无权限的菜单项不显示，子菜单全部无权限时父级也不显示。

### 7.3 折叠状态

* 左侧菜单支持折叠（宽度从 `240px` 变为 `64px`），图标 + `title` 隐藏。

* 折叠状态保存在 `localStorage` 或 pinia 中，保持用户偏好。

### 7.4 当前路由高亮

* `el-menu` 绑定 `:default-active="$route.path"`，当路由变化时自动高亮匹配项。

* 对于动态路由（如 `/case/detail/:id`），需设置 `index` 为父级路径或使用 `resolve` 方法。

9. 禁止事项（AI 或开发者必须遵守）

--------------------

* ❌ 禁止在任何 `.vue` 文件的 `<style scoped>` 中定义全局 CSS 变量。

* ❌ 禁止在组件内硬编码颜色（如 `color: red`、`background: #fff`）。

* ❌ 禁止使用 `!important` 覆盖 Element Plus 默认样式（应通过 CSS 变量修改）。

* ❌ 禁止使用非 Element Plus 的栅格系统（如 Bootstrap、自写 flex 网格）。

* ❌ 禁止在菜单栏中使用 `el-menu` 以外的自定义列表模拟菜单。

* ❌ 禁止在深色主题下遗漏任何文本或背景变量，导致不可读。

* * *

10. AI 生成代码时的检查清单

-----------------

当你（AI）生成任何新页面或组件时，必须确认：

1. ✅ 布局使用 `el-container` / `el-row` / `el-col` 或 `flex`（基于 CSS 变量间距）。

2. ✅ 所有颜色值均通过 `var(--xxx)` 引用。

3. ✅ 组件内部 `<style scoped>` 只包含布局、定位、特定组件尺寸等，不包含色值。

4. ✅ 深色主题测试：添加 `html.dark` 类后页面依然可读。

5. ✅ 菜单配置与路由保持一致，支持递归渲染 `children`。

6. ✅ 页面容器背景为 `var(--bg-page)`，内容卡片背景为 `var(--bg-container)`。
