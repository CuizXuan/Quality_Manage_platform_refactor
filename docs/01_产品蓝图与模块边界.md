# 产品蓝图与模块边界

## 1. 产品定位

Quality Manage Platform 是一个现代化质量管理平台。它不是单一接口调试工具，而是一套覆盖接口调试、测试资产沉淀、自动化执行、缺陷闭环、质量门禁和 AI 辅助分析的质量工程系统。

当前代码底座只包含：

- 登录页
- JWT 认证
- 用户管理
- 角色权限
- 组织管理
- 菜单管理
- 深色与浅色主题

后续业务模块必须基于这个底座开发。

## 2. 模块总览

### 系统管理

目标：维护平台自身的用户、角色、组织、菜单和权限。

功能：

- 用户增删改查
- 用户状态启停
- 用户绑定组织
- 用户绑定角色
- 角色增删改查
- 角色绑定权限
- 组织增删改查
- 菜单增删改查
- 菜单权限编码配置

### 工作台

目标：展示质量状态和快捷入口。

功能：

- 当前项目概览
- 当前环境概览
- 最近调试记录
- 最近执行记录
- 最近失败项
- 最近缺陷
- 质量门禁状态
- AI 建议摘要

### 终端调试台

目标：承接接口调试，是测试资产进入系统的第一入口。

功能：

- 手工编辑请求
- 粘贴 cURL / Fetch / 原始 HTTP 请求解析
- 编辑 Method、URL、Query、Header、Cookie、Auth、Body
- 支持 JSON、Form、Multipart、Raw
- 发送请求
- 查看响应状态、耗时、响应头、响应体
- 响应 JSON 格式化与搜索
- 保存调试历史
- 收藏请求
- 保存为基础用例
- 从响应生成断言
- 追加为场景步骤

### 用例中心

目标：管理可复用的接口测试资产。

功能：

- 用例增删改查
- 文件夹管理
- 标签管理
- 用例复制
- 用例版本
- 请求配置维护
- 断言维护
- 变量维护
- 前后置脚本
- 从终端保存基础用例
- 从基础用例派生变体
- 批量执行

### 用例变体

目标：把一条基础用例拆解成不同测试情况。

功能：

- 正常值变体
- 边界值变体
- 空值变体
- 缺失字段变体
- 类型错误变体
- 枚举非法变体
- 超长字段变体
- 鉴权失败变体
- 权限不足变体
- 返回结构校验变体
- 返回业务值校验变体
- 性能阈值变体

### 场景编排

目标：把多个用例和变体组织成业务链路。

功能：

- 场景增删改查
- 步骤增删改查
- 从用例中心选择步骤
- 从终端调试追加步骤
- 步骤排序
- 步骤启停
- 失败策略
- 重试策略
- 超时配置
- 步骤间变量提取和注入
- 场景版本
- 场景执行

### 环境与变量

目标：统一维护不同环境下的执行上下文。

功能：

- 环境增删改查
- 基础 URL
- 默认 Header
- 鉴权配置
- 全局变量
- 项目变量
- 环境变量
- 用例变量
- 场景变量
- 敏感变量脱敏

### 执行历史

目标：记录终端、用例和场景的每次执行。

功能：

- 执行记录查询
- 请求快照
- 响应快照
- 断言结果
- 步骤日志
- 失败原因
- 重试记录
- 与上次结果对比
- 创建缺陷
- 进入报告

### 报告中心

目标：把执行历史沉淀为质量报告。

功能：

- 报告生成
- 报告列表
- 报告详情
- 通过率
- 失败分布
- 耗时统计
- 变体覆盖
- 场景覆盖
- 缺陷关联
- 分享与下载

### 缺陷中心

目标：让执行失败进入闭环。

功能：

- 缺陷增删改查
- 从执行失败创建缺陷
- 关联用例、变体、场景、报告
- 状态流转
- 严重程度
- 优先级
- 修复版本
- 复测结果
- 外部系统同步

### 质量门禁

目标：为发布提供自动放行判断。

功能：

- 门禁规则增删改查
- 通过率门禁
- 失败率门禁
- 高危缺陷门禁
- 覆盖率门禁
- 性能门禁
- 阻断、警告、放行
- 人工豁免

### AI 中枢

目标：提供生成、分析、归因和建议能力。

功能：

- 从接口生成用例
- 从基础用例生成变体
- 从响应生成断言
- 从失败结果分析原因
- 从报告生成质量总结
- 自愈建议
- 模型配置
- Prompt 模板管理
- AI 调用历史
