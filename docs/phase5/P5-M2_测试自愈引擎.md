# P5-M2: 测试自愈引擎

## 1. 模块概述

**功能**: 自动检测并修复因非代码缺陷导致的测试失败

**可自愈场景**:
| 失败类型 | 自愈策略 | 置信度 |
|----------|----------|--------|
| 元素定位失败 | AI 重新生成定位器 | 高 |
| 断言值变化 | 分析响应结构，调整期望值 | 中 |
| 超时 | 增加等待时间/重试次数 | 高 |
| 环境问题 | 切换到备用环境 | 中 |
| 数据依赖缺失 | 自动创建测试数据 | 中 |
| 认证过期 | 自动刷新 Token | 高 |

## 2. 数据库设计

### 2.1 自愈日志表 `self_heal_log`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 自增主键 |
| failure_log_id | INTEGER | FK | 关联的失败执行记录 |
| heal_action | VARCHAR(50) | | 自愈动作类型 |
| heal_config | TEXT | | 自愈配置 JSON |
| heal_success | BOOLEAN | | 是否自愈成功 |
| before_snapshot | TEXT | | 修复前快照 |
| after_snapshot | TEXT | | 修复后快照 |
| confidence | FLOAT | | 置信度 0-1 |
| ai_reasoning | TEXT | | AI 推理过程 |
| human_approved | BOOLEAN | | 是否人工确认 |
| created_at | TIMESTAMP | DEFAULT | 创建时间 |

**自愈动作类型**:
| 类型 | 说明 | 示例 |
|------|------|------|
| update_locator | 更新元素定位器 | 自动更新 CSS/XPath |
| update_assertion | 更新断言值 | 自动调整期望值 |
| add_wait | 增加等待时间 | 自动添加等待 |
| retry_with_backoff | 重试并退避 | 自动重试配置 |
| skip_step | 跳过失败步骤 | 标记为非关键步骤 |

## 4. 服务层设计

### 4.1 `SelfHealService.py`

```python
class SelfHealService:
    """测试自愈服务"""
    
    HEAL_ACTIONS = {
        'update_locator': '更新元素定位器',
        'update_assertion': '更新断言值',
        'add_wait': '增加等待时间',
        'retry_with_backoff': '重试并退避',
        'skip_step': '跳过失败步骤'
    }
    
    def classify_failure(self, failure_log_id: int) -> dict:
        """失败分类"""
        
    def match_heal_strategy(self, failure_type: str, context: dict) -> list:
        """匹配自愈策略"""
        
    def execute_heal(self, failure_log_id: int, strategy: dict, auto_approve: bool) -> dict:
        """执行自愈"""
        
    def verify_heal(self, heal_id: int) -> bool:
        """验证自愈结果"""
        
    def rollback_heal(self, heal_id: int):
        """回滚自愈"""
```
