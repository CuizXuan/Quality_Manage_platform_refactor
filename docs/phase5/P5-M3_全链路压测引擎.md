# P5-M3: 全链路压测引擎

## 1. 模块概述

**功能**: 支持生产流量录制与回放，实现真实场景压测

**核心能力**:
- 流量录制 (Nginx/Envoy/K8s Sidecar)
- 流量回放 (时间轴精确回放/并发放大)
- 流量染色 (标记压测流量)
- 影子环境隔离
- Diff 对比分析

## 2. 数据库设计

| id | INTEGER | PK, AUTO | 自增主键 |

| traffic_data | TEXT | | 流量数据存储路径 |
| request_count | INTEGER | | 请求总数 |

| time_range_start | TIMESTAMP | | 开始时间 |

## 3. API 设计

### 3.1 流量录制

```
POST /api/traffic/record
Request:
{
  "name": "生产流量录制-20260111",
  "source": "kubernetes",
  "filter_rules": {
    "exclude_paths": ["/health", "/metrics"],
    "include_methods": ["GET", "POST", "PUT"]
  },
  "environment_id": 1,
  "duration_minutes": 60
}

Response:
{
  "record_id": 123,
  "status": "recording",
  "start_time": "2026-01-11T10:00:00Z"
}
```

### 3.2 流量回放

```
POST /api/traffic/replay
Request:
{
  "record_id": 123,
  "config": {
    "concurrency": 10,
    "duration_seconds": 300,
    "think_time_ms": 100
  },
  "target_environment_id": 2,
  "enable_shadow": true
}

Response:
{
  "replay_id": 456,
  "status": "running"
}
```

## 4. 服务层设计

### 4.1 `TrafficRecordService.py`

```python
class TrafficRecordService:
    """流量录制服务"""
    
    def start_recording(self, config: dict) -> dict:
        """开始录制"""
        
    def stop_recording(self, record_id: int):
        """停止录制"""
        
    def get_recording_status(self, record_id: int) -> dict:
        """获取录制状态"""
        
    def export_traffic(self, record_id: int) -> dict:
        """导出流量数据"""
```

### 4.2 `TrafficReplayService.py`

```python
class TrafficReplayService:
    """流量回放服务"""
    
    def start_replay(self, record_id: int, config: dict) -> dict:
        """开始回放"""
        
    def stop_replay(self, replay_id: int):
        """停止回放"""
        
    def get_replay_progress(self, replay_id: int) -> dict:
        """获取回放进度"""
        
    def add_loadtest_headers(self, request: dict) -> dict:
        """添加压测标记头"""
```

class DiffEngineService:
