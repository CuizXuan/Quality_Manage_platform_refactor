# 用例管理模块页面设计方案

## 1. 需求概述

质量管理平台需要统一的用例管理模块，支持两类用例：

- **接口用例**：针对单个 API 的测试，配置请求参数、头信息、Body、断言等。
- **功能用例**：面向业务流程的场景测试，包含前置条件、执行步骤、预期结果、用例等级等。

两类用例有公共属性（名称、描述、所属模块、标签等），也有各自特有字段。页面需要支持灵活的新增、编辑、复制、删除、导入导出、筛选等功能。

## 2. 页面布局方案

采用 **三栏布局**（左侧目录树 + 中间用例列表 + 右侧详情表单），这是后台测试用例管理的高效模式。

### 2.1 整体布局结构

text

```
+----------------------------------------------------------------------------------+
|  用例管理                                                    [导入] [导出] [帮助]  |
+----------------------------------------------------------------------------------+
|         |                      |                                                |
| 左侧目录树 │   中间用例列表          │   右侧详情编辑区                             |
| (模块/项目)│  (支持筛选、搜索)       │  (动态表单，根据类型切换字段)                 |
|         |                      |                                                |
|  ▼ 项目A   │ □ ID │ 名称 │类型│等级│操作  │   [保存] [复制] [删除] [执行]            |
|    ▼ 模块1 │  1   │ 登录 │接口│ P0 │ ... │   ───────────────────────────────      |
|      - 用例1│  2   │ 下单 │功能│ P1 │ ... │   公共字段                             |
|      - 用例2│      │      │    │    │     │   用例名称: [________]                 |
|    ▼ 模块2 │      │      │    │    │     │   所属模块: [下拉]      类型: ⚫接口 ○功能|
|      - 用例3│      │      │    │    │     │   等级: [P0▼]  标签: [________]       |
|         |                      │     │   前置条件: [___________________]           |
|         |                      │     │   描述: [___________________]              |
|         |                      │     │   ───────────────────────────────          |
|         |                      │     │   [接口用例特有字段] 或 [功能用例特有字段]    |
|         |                      │     │   (根据类型动态显示)                         |
|         |                      │     │                                             |
|         |                      │     └────────────────────────────────────────────┘
+----------------------------------------------------------------------------------+
```



### 2.2 各区域详细设计

#### 2.2.1 左侧目录树

- 展示项目/模块层级，支持无限深度。
- 每个节点显示用例数量（可选）。
- 右键菜单：新建模块、重命名、删除、移动。
- 支持拖拽排序、拖拽移动用例到不同模块。
- 顶部有“全部用例”根节点，点击显示所有用例。

#### 2.2.2 中间用例列表

- 表格列：序号、用例名称、用例类型（标签）、优先级、最后修改时间、操作（编辑、复制、删除、执行）。
- 顶部工具栏：
  - 搜索框（按名称/描述模糊搜索）。
  - 类型筛选（全部/接口用例/功能用例）。
  - 等级筛选（P0/P1/P2/P3）。
  - “新建用例”按钮（点击后右侧表单清空，进入新增模式，默认选中当前目录模块）。
- 支持多选用例，批量删除、批量修改等级、批量导出。
- 分页器（默认每页20条）。

#### 2.2.3 右侧详情编辑区

动态表单，根据“用例类型”切换显示不同字段。顶部有操作按钮：保存、复制（保存为新用例）、删除、执行（如果是接口用例直接发送请求并显示响应；功能用例则弹出执行窗口）。

**公共字段**（两种类型共有）：

| 字段名   | 组件                      | 校验/说明                               |
| :------- | :------------------------ | :-------------------------------------- |
| 用例名称 | el-input                  | 必填，最大100字符                       |
| 所属模块 | el-cascader / tree-select | 从左侧目录树同步，必填                  |
| 用例类型 | el-radio-group            | 接口用例 / 功能用例，切换时重置特有字段 |
| 优先级   | el-select                 | P0(最高) / P1 / P2 / P3                 |
| 标签     | el-select(tags)           | 可多选，支持自定义标签                  |
| 前置条件 | el-input textarea         | 富文本或普通文本                        |
| 描述     | el-input textarea         | 补充说明                                |

**接口用例特有字段**（当类型=接口时显示）：

| 字段名   | 组件                        | 说明                                           |
| :------- | :-------------------------- | :--------------------------------------------- |
| 请求方法 | el-select (GET/POST/...)    |                                                |
| URL      | el-input                    | 支持环境变量 {{base_url}}                      |
| 请求头   | 键值对表格 (可动态增删)     | 每行：参数名、参数值、描述                     |
| 请求参数 | 同上（Query参数）           | 可切换 Params / Body                           |
| Body类型 | el-radio-group              | none / x-www-form-urlencoded / raw / form-data |
| Body内容 | el-input / code-editor      | raw时支持JSON/XML/Text，可格式化校验           |
| 认证配置 | el-collapse 折叠面板        | Basic Auth / Bearer Token / API Key            |
| 断言规则 | 表格 (字段、操作符、期望值) | 支持状态码、响应体字段、JSONPath等             |

**功能用例特有字段**（当类型=功能时显示）：

| 字段名   | 组件                   | 说明                                           |
| :------- | :--------------------- | :--------------------------------------------- |
| 执行步骤 | 步骤列表（可拖拽排序） | 每步包含：步骤描述、预期结果、测试数据（JSON） |
| 测试数据 | 表格 / JSON编辑器      | 全局测试数据集（用于数据驱动）                 |
| 后置动作 | 文本域                 | 清理数据、重置状态等                           |

### 2.3 按钮布局与交互

| 按钮位置     | 按钮名 | 触发行为                                                     |
| :----------- | :----- | :----------------------------------------------------------- |
| 顶部工具栏   | 导入   | 支持 Excel / Postman / Swagger 导入，弹窗选择覆盖策略        |
|              | 导出   | 导出选中或当前筛选后的用例（Excel/JSON）                     |
| 列表行内     | 编辑   | 加载该用例到右侧表单，进入编辑模式                           |
|              | 复制   | 弹出复制对话框，新名称默认为“原名称 - 副本”，保存后刷新列表  |
|              | 删除   | 软删除，二次确认                                             |
|              | 执行   | 接口用例：发送请求，右侧下方或侧边弹窗显示响应结果；功能用例：打开执行悬浮窗，实时显示步骤进度 |
| 右侧表单顶部 | 保存   | 新增或更新用例，保存后刷新列表并清空表单（可选留在当前页）   |
|              | 另存为 | 将当前配置保存为新用例，需要输入新名称                       |
|              | 删除   | 删除当前编辑的用例，成功后清空表单并刷新列表                 |
|              | 执行   | 同列表行内执行                                               |

### 2.4 响应展示区（接口用例）

当执行接口用例时，在右侧详情区下方或右侧抽屉显示：

- 状态码、耗时、响应大小。
- 响应体：语法高亮（JSON/XML/HTML），支持折叠、美化、复制。
- 断言结果：显示是否通过，失败原因。

## 3. 交互细节与规范

### 3.1 用例类型切换行为

- 新建时默认为“接口用例”，用户可手动切换。
- 切换类型时，若当前表单已有内容，弹出确认：“切换类型将清除当前特有字段数据，是否继续？”
- 保存后类型不可更改（可设计为需要复制后修改类型），避免历史数据混乱。

### 3.2 批量操作

- 列表支持复选框，批量删除、批量修改优先级、批量移动模块。
- 批量导出时生成压缩包（内含JSON文件或单个Excel）。

### 2.2 操作说明

| 操作       | 交互方式                                      |
| :--------- | :-------------------------------------------- |
| 新建根模块 | 弹窗输入模块名称，parentId=0                  |
| 新增子模块 | 点击节点右侧的 `[+]` 按钮，弹窗输入子模块名称 |
| 编辑模块   | 双击模块名称或点击 `[编辑]`，弹出修改框       |
| 删除模块   | 二次确认，若模块下有用例则提示“是否迁移用例”  |
| 移动模块   | 拖拽节点到其他节点上，或右键菜单“移动到...”   |
| 排序       | 拖拽调整同级顺序，保存后更新 sort_order       |
| 导入/导出  | Excel格式导入导出模块树结构                   |

### 2.3 用例迁移规则

- 删除模块时，如果该模块下存在用例，必须选择：
  - 将用例移动到其他指定模块（弹出模块树选择器）。
  - 或级联删除用例（慎用，需二次确认）。

## 三、用例归属到模块的方式

### 3.1 新建/编辑用例时

右侧详情表单中的“所属模块”字段是一个树形选择器，样式如下：

text

```
所属模块:  [用户中心 / 登录模块 / 账号密码登录 ▼]
```



用户点击下拉框，展示完整模块树，支持搜索、展开折叠，选择后保存 `module_id`。

### 3.2 批量移动用例

在中间用例列表中，勾选多个用例后，点击工具栏的“批量移动”按钮，弹出模块树选择器，将所选用例一次性移动到目标模块。

### 3.3 拖拽移动

用户可以直接将左侧目录树中的用例节点拖拽到另一个模块节点上，实现快速归属变更（适合少量操作）。

## 4. 技术实现建议（遵循现有规范）

- **UI 组件**：Element Plus（el-table, el-tree, el-form, el-tabs 等）。
- **动态表单**：使用 `v-if` 根据 `caseType` 切换不同字段组，封装为 `<ApiCaseForm>` 和 `<FuncCaseForm>` 子组件。
- **状态管理**：`useCaseStore` 管理当前选中的用例、目录树、列表数据。
- **样式**：遵循已有的 CSS 变量，支持深色主题，左右分栏使用 `el-container` + `el-aside` + `el-main`。
- **响应式**：左侧目录树宽度 280px，中间列表最小宽度 600px，右侧表单宽度 480px（可折叠）。

## 6. 总结

该设计方案统一了两种用例的管理入口，利用左侧树形目录组织模块，中间列表快速筛选与批量操作，右侧动态表单按类型精确编辑。既满足了接口调试的细粒度配置，也兼顾了业务场景的步骤化管理，同时保持界面整洁、操作流畅。后续可扩展用例执行历史、测试计划关联等高级功能。
