# AI 功能实现规范

## 1. AI 能力边界

AI 只做辅助生成和建议，不能直接替用户修改核心资产，除非用户明确采纳。

必须支持“建议 -> 预览 -> 采纳 -> 写入”的流程。

## 2. AI 模块

### AI 用例生成

输入：

- OpenAPI 文档
- 调试请求
- 需求文本
- 接口响应示例

输出：

- 基础用例草稿
- 推荐断言
- 推荐标签
- 风险说明

### AI 变体生成

输入：

- 基础用例
- 参数结构
- 响应结构
- 历史执行结果

输出：

- 边界值变体
- 异常值变体
- 权限类变体
- 返回值校验变体
- 性能阈值建议

### AI 断言生成

输入：

- 响应体
- 响应头
- 状态码
- OpenAPI Schema

输出：

- 状态码断言
- JSONPath 断言
- Schema 断言
- 业务字段断言

### AI 失败归因

输入：

- 请求快照
- 响应快照
- 断言结果
- 执行日志
- 历史成功结果

输出：

- 失败类型
- 疑似原因
- 影响范围
- 建议修复方向
- 是否建议创建缺陷

### AI 报告总结

输入：

- 执行汇总
- 失败明细
- 缺陷列表
- 趋势数据

输出：

- 质量摘要
- 风险点
- 阻断建议
- 后续测试建议

## 3. AI 数据模型

### AIModelConfig

字段：

- id
- provider
- model
- base_url
- api_key_encrypted
- enabled
- default_for

### AITask

字段：

- id
- task_type
- input_snapshot
- output_result
- status
- error_message
- created_by
- created_at

### AISuggestion

字段：

- id
- task_id
- suggestion_type
- target_type
- target_id
- payload
- status
- accepted_at
- accepted_by

## 4. Prompt 规范

Prompt 必须包含：

- 任务目标
- 输入上下文
- 输出 JSON Schema
- 禁止编造字段
- 不确定时返回 `confidence`

输出必须是结构化 JSON，不允许只返回自然语言。

## 5. 安全规范

- API Key 必须加密存储
- Prompt 中不得包含明文敏感变量
- AI 输出不得自动执行
- AI 输出必须可追溯到输入快照
- AI 建议必须记录采纳人和采纳时间
