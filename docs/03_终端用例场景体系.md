# 终端、用例、场景体系

## 1. 设计原则

终端调试不是临时工具，而是测试资产生产线的入口。

正确链路：

1. 终端调试接口
2. 保存为基础用例
3. 基础用例派生测试变体
4. 变体加入场景
5. 场景执行
6. 执行结果进入报告、缺陷、门禁

## 2. 终端调试

### 请求编辑

必须支持：

- Method
- URL
- Query
- Header
- Cookie
- Auth
- Body
- 环境变量
- 原始请求解析

### 响应查看

必须支持：

- Status Code
- Duration
- Response Headers
- Response Body
- Error Message
- JSON 格式化
- 搜索
- 复制

### 保存为用例

保存内容：

- 请求配置
- 最近响应
- 基础断言建议
- 当前环境
- 来源调试记录

保存模式：

- 新建基础用例
- 覆盖已有用例
- 保存为新版本
- 保存为已有用例变体

## 3. 用例变体

### 变体类型

| 类型 | 说明 |
|------|------|
| normal | 正常成功路径 |
| boundary | 边界值 |
| empty | 空值 |
| missing | 缺失字段 |
| type_error | 类型错误 |
| enum_error | 枚举非法 |
| length_error | 超长字段 |
| auth_error | 鉴权失败 |
| permission_error | 权限不足 |
| response_schema | 返回结构校验 |
| response_value | 返回业务值校验 |
| performance | 性能阈值 |

### 变体继承规则

变体继承基础用例：

- method
- url
- headers
- cookies
- auth_config
- body_type

变体可以覆盖：

- query_params
- headers
- body
- expected_status
- assertions
- timeout_ms

## 4. 断言体系

基础断言：

- 状态码等于
- 响应耗时小于
- 响应体包含字段
- 响应体字段等于
- 响应体字段类型正确
- 响应头包含

高级断言：

- JSONPath
- 正则
- JSON Schema
- 数组长度
- 字段范围
- 自定义脚本

断言来源：

- 手工配置
- 响应自动推导
- OpenAPI Schema 推导
- 历史成功响应推导
- AI 建议采纳

## 5. 场景步骤

场景步骤引用用例或变体，不复制完整用例。

字段：

- step_name
- case_id
- variant_id
- enabled
- sort_order
- retry_count
- timeout_ms
- failure_strategy
- extract_rules
- inject_rules

失败策略：

- stop
- continue
- retry
- skip

## 6. 步骤间变量传递

提取来源：

- 响应体 JSONPath
- 响应头
- Cookie
- 状态码
- 自定义脚本

注入目标：

- URL Path
- Query
- Header
- Cookie
- Body
- Assertion

示例：

登录接口返回 `$.data.token`，提取为 `auth_token`，后续接口使用 `Authorization: Bearer {{auth_token}}`。
