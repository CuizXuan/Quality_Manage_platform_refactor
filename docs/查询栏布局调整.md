### . 整体布局方式

* **组件**：`el-form` + `el-row` / `el-col`（栅格系统）

* **布局模式**：将 `el-form` 的 `inline` 属性设为 `false`（默认），利用栅格实现响应式多列布局

* **间距控制**：`el-row` 的 `gutter` 属性设置列之间的水平间隔

### 2. 响应式断点策略

* **属性**：`el-col` 的 `xs`、`sm`、`md`、`lg`、`xl` 属性

* **分配规则**：
  
  * 超小屏（手机）：每个条件占满整行（`span=24`）
  
  * 小屏（平板）：每行显示 2 个条件（`span=12`）
  
  * 中屏（笔记本）：每行显示 3 个条件（`span=8`）
  
  * 大屏（桌面）：每行显示 4 个条件（`span=6`）

* **好处**：无需媒体查询，自动适配不同设备

### 3. 查询条件过多时的折叠方案

* **组件**：自定义按钮或 `el-link` 触发折叠/展开

* **状态管理**：组件内响应式变量控制额外条件的显示与隐藏（`v-if` 或 `v-show`）

* **布局技巧**：默认只显示 2～3 个常用条件，其余条件放在第二行或更多区域内

### 4. 按钮位置与对齐

* **按钮容器**：在栅格中单独分配一列（最后一列），通过 `flex` 或文本对齐实现右对齐

* **对齐方式**：给按钮所在的 `el-col` 添加自定义样式 `display: flex; justify-content: flex-end;`

* **按钮组合**：查询（主按钮）、重置（普通按钮）、高级筛选（链接按钮）排列有序

* **批量操作按钮**：独立放在表格上方左侧，与查询栏用 `margin` 或 `el-divider` 视觉分隔

表格列宽的精细控制方案
-----------

### 1. 列宽基础策略

* **固定宽度列**：使用 `width` 属性，适用于内容长度变化小、需对齐的列（如 Method、优先级、操作按钮组）

* **弹性列**：使用 `min-width` 属性，不设 `width`，表格会根据剩余空间按比例分配宽度，保证每列最小宽度

* **内容自适应列**：设置 `table-layout="auto"`，但通常不推荐（性能较差，宽度不可控），优先使用 `min-width`

### 2. 长文本处理

* **超出省略**：`show-overflow-tooltip` 属性，当内容超过列宽时显示省略号，鼠标悬浮展示完整文本

* **适用列**：URI、用例名称、描述等长度不可控的字段

### 3. 列宽交互增强

* **用户可调整**：`el-table` 开启 `resizable` 属性（默认开启），用户可拖拽列边界改变宽度

* **保存用户偏好**：
  
  * 监听 `header-dragend` 事件，获取新的列宽
  
  * 将列标识（如 `column.property`）与宽度值存入 `localStorage` 或发送到后端
  
  * 页面初始化时读取存储值，动态设置对应列的 `width` 属性

* **列宽重置**：提供“重置列宽”按钮，清除存储并恢复默认宽度

### 4. 响应式隐藏/显示列

* **工具**：`vueuse` 的 `useWindowSize` 或原生 `matchMedia`

* **逻辑**：根据屏幕宽度计算布尔值，使用 `v-if` 控制某些列是否渲染

* **示例规则**：
  
  * 宽度 < 768px：隐藏“创建人”列
  
  * 宽度 < 992px：隐藏“更新时间”列
  
  * 宽度 < 1200px：合并部分元信息列

### 5. 操作列固定

* **属性**：`fixed="right"`，让操作列始终显示在表格右侧，不随横向滚动条滚动

* **宽度**：操作列建议使用固定宽度（如 150～180px），确保按钮不会被压缩换行

### 6. 表头与列的对齐

* **对齐属性**：`align="center"` 用于简短字段（Method、优先级），`align="left"` 用于文本内容（用例名称、URI）

* **表头对齐**：通过 `header-align` 单独控制表头对齐方式

* * *

辅助增强方案
------

### 1. 查询栏与表格之间的间距控制

* 使用 `margin-bottom` 或 `padding` 统一设计变量（如 16px / 20px）

### 2. 表格高度与滚动区域

* 当表格行数较多时，设置 `max-height`，配合 `el-table` 的 `height` 或 `max-height` 属性生成固定表头的滚动区域

### 3. 加载状态与空状态

* 查询栏点击查询时，设置表格的 `v-loading` 指令

* 无数据时，`el-table` 自动显示 `empty` 插槽或默认文案，可自定义

### 4. 整体风格统一

* 查询栏与表格使用相同的圆角、阴影、边框变量（Element Plus 主题变量）

* 按钮间距使用 `margin-left: 12px` 或 `el-button` 自带间距

总结（所用技术/属性/模式一览）
----------------

| 控制项        | 使用的 Element Plus 组件 / 属性 / 模式                                        |
| ---------- | -------------------------------------------------------------------- |
| **查询栏布局**  | `el-form`, `el-row`, `el-col` (栅格), `gutter`, `xs/sm/md/lg/xl` 响应式断点 |
| **条件折叠**   | 响应式变量 + `v-if` / `v-show`，`el-link` 或自定义按钮                           |
| **按钮对齐**   | 栅格最后一列 + `flex` 布局 (`justify-content: flex-end`)                     |

| **表格弹性列宽** | `min-width` 属性，不设 `width`                                            |
| **固定宽度列**  | `width` 属性（数字，单位 px）                                                 |
| **长文本省略**  | `show-overflow-tooltip` 属性                                           |
| **列宽拖拽**   | `resizable` 属性（表格级），`header-dragend` 事件                              |
| **保存列宽偏好** | `localStorage` / 后端接口 + 动态绑定 `width`                                 |
| **响应式隐藏列** | `useWindowSize` + `v-if`                                             |
| **操作列固定**  | `fixed="right"` 属性                                                   |
| **表格滚动**   | `max-height` / `height` 属性                                           |
| **加载状态**   | `v-loading` 指令                           |
