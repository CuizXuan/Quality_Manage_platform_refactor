# 后端开发规范

## 1. 技术栈

- FastAPI
- SQLAlchemy
- Pydantic v2
- SQLite 开发环境
- PostgreSQL 生产环境预留

## 2. 目录结构

```text
backend/app/
├── main.py
├── config.py
├── database.py
├── models/
├── schemas/
├── routers/
├── services/
└── repositories/
```

当前脚手架暂未创建 `repositories/`，业务模块增加复杂查询时必须补上。

## 3. 分层职责

### Router

只负责：

- 接收参数
- 注入依赖
- 调用 Service
- 返回响应

不得写复杂业务逻辑。

### Service

负责：

- 业务规则
- 状态流转
- 跨表操作
- 外部调用

### Repository

负责：

- 数据库查询
- 分页
- 复杂筛选
- 聚合统计

## 4. API 规范

REST 风格：

- `GET /api/cases`
- `POST /api/cases`
- `GET /api/cases/{id}`
- `PUT /api/cases/{id}`
- `DELETE /api/cases/{id}`

动作类接口：

- `POST /api/terminal/debug`
- `POST /api/terminal/save-as-case`
- `POST /api/cases/{id}/run`
- `POST /api/scenarios/{id}/run`
- `POST /api/ai/case-variants`

分页响应：

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "page_size": 20
}
```

## 5. 权限规范

每个菜单和操作都要有权限编码。

格式：

```text
resource:action
```

示例：

- `user:manage`
- `case:view`
- `case:manage`
- `scenario:run`
- `ai:generate`

## 6. 数据库规范

- 模型文件按领域拆分
- 字段命名使用 snake_case
- 所有核心表必须有 `created_at`
- 重要资产必须有 `updated_at`
- 业务数据必须有 `created_by`
- 状态字段使用明确枚举字符串
- JSON 字段只存结构化配置，不存大段日志

## 7. 测试规范

核心链路必须有测试：

- 登录
- 用户 CRUD
- 角色权限
- 终端保存用例
- 用例派生变体
- 场景执行
- 执行历史快照
- AI 建议采纳
