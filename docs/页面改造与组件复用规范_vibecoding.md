# 页面改造与组件复用规范（VibeCoding 版）

## 1. 目标

本文档用于指导质量管理平台后续模块改造，核心目标是：

- 保持页面风格统一
- 保持布局和交互一致
- 降低 vibecoding 时的返工成本
- 将用例管理中可复用的样式和组件沉淀为公共层

## 2. 适用范围

适用于：

- 列表页
- 查询页
- 配置页
- 表单页
- 弹窗页
- 调试台保存类页面

不强制包含：

- 分类树
- 统计卡片
- 侧栏
- 高级配置区

这些模块按业务需要出现，不是固定结构。

## 4. 页面骨架

推荐页面骨架：

1. 标题区
2. 可选统计区
3. 可选分类区
4. 查询区
5. 列表区
6. 弹窗区

没有的区块直接删除，不要硬塞。

### 4.1 标准列表页强约束

后续模型改造列表型页面时，必须先判断页面属于哪一种结构：

| 页面类型 | 必须包含 | 可选包含 |
| --- | --- | --- |
| 基础列表页 | 标题区、查询区、列表区、分页、弹窗 | 统计区 |
| 分类列表页 | 标题区、左侧分类区、查询区、列表区、分页、弹窗 | 统计区 |
| 资产管理页 | 标题区、查询区、列表区、批量操作、弹窗 | 分类区、统计区 |
| 配置管理页 | 标题区、查询区、列表区、弹窗 | 统计区 |

重要规则：

- 没有分类树时，右侧工作区必须直接占满全宽，不要保留空的左栏。
- 没有统计区时，标题区下方直接接查询区，不要硬塞指标卡。
- 没有复杂字段时，弹窗不要强行做高级信息区。
- 页面仍然要保留统一背景、面板质感、字体、表格透明度和查询栏对齐规则。

## 5. 尺寸规范

### 5.1 页面

- 页面内边距：`12px`
- 区块间距：`12px`
- 页面高度：`100%`
- 子容器最小高度：`0`
- 面板圆角：`8px`

### 5.2 字体

- 页面正文：`14px`
- 表单标签：`14px`
- 表格正文：`14px`
- 表格表头：`13px`
- 弹窗标题：`16px`
- 区块标题：`15px`
- 辅助说明：`12px ~ 13px`

### 5.3 按钮

- 主按钮高度：`34px`
- 次按钮高度：`34px`
- 表格内操作按钮：`12px ~ 13px`
- 主按钮优先使用主色或品牌渐变

### 5.4 表格

- 表头高：`44px`
- 行高：`48px`
- 固定列：只固定关键列
- 长文本：省略 + tooltip

### 5.5 弹窗

- 简单弹窗：`760px`
- 标准弹窗：`960px`
- 复杂弹窗：`width: min(1280px, 95vw)`
- 内容区高度：`70vh ~ 80vh`
- 顶部间距：`3vh ~ 6vh`

## 6. 字体规范

- 主内容字体：`14px`
- 表格标题字体：`13px`
- 表格内容字体：`14px`
- 弹窗标题字体：`16px`
- 说明文字：`12px ~ 13px`

要求：

- 同一页面不要混用过多字号。
- 不要把正文做得过小。
- 重点字段要比辅助字段更清晰。

## 7. 背景与质感

推荐背景结构：

1. 主题底色
2. 细网格底纹
3. 轻扫描线
4. 极淡节点流线

要求：

- 背景弱于正文
- 深浅色都要适配
- 动效轻，不打扰阅读
- 优先使用 CSS / 伪元素，不依赖真实图片

当前公共实现已抽象到：

- `.page-shell`
- `.page-shell--tech-grid`
- `.page-surface`
- `.page-surface--glass`
- `.page-surface--tech`

### 7.1 背景流动感强约束

用例管理里的“现代科技感”不是单纯深色背景，必须同时具备以下层次：

1. 页面底层：深浅色主题底色。
2. 页面网格：`28px * 28px` 的细网格。
3. 页面扫描线：横向轻微移动，动画时间约 `14s`。
4. 点阵漂移：极淡节点点阵，动画时间约 `18s`，只做轻微位移。
5. 面板内部纹理：查询区、分类区、列表区内部也要有细网格，不允许被实心面板完全盖住。
6. 表格内部纹理：表格区域必须能透出轻网格和扫描质感。

必须使用：

- 页面外壳：`.page-shell--tech-grid`
- 面板：`.page-surface--glass` 或 `.page-surface--tech`
- 表格面板：参考 `CaseTable.vue` 的 `.case-table-panel`

禁止：

- 只设置纯色背景。
- 只在页面外层做背景，表格和面板内部完全不透明。
- 使用高强度全屏粒子动画。
- 使用真实背景图代替 CSS 网格和扫描线。

### 7.2 面板透明度规则

面板要“有玻璃感，但不影响阅读”。

深色主题建议：

- 面板底色：`rgba(20, 22, 27, 0.36)` 到 `rgba(20, 22, 27, 0.58)`
- 面板渐变：`rgba(15, 23, 42, 0.54)` 到 `rgba(15, 23, 42, 0.34)`
- 边框：`rgba(56, 189, 248, 0.18)`
- 内部网格透明度：`rgba(56, 189, 248, 0.05)` 左右
- 玻璃模糊：`backdrop-filter: blur(18px) saturate(1.2)`

浅色主题建议：

- 面板底色：`rgba(255, 255, 255, 0.62)` 到 `rgba(255, 255, 255, 0.76)`
- 面板渐变：`rgba(255, 255, 255, 0.76)` 到 `rgba(245, 250, 255, 0.58)`
- 边框：`rgba(22, 119, 255, 0.14)`
- 内部网格透明度：`rgba(22, 119, 255, 0.04)` 左右

如果内容读不清，优先提高文字对比度和行背景透明度，不要直接删除背景纹理。

## 9. 查询栏规范

推荐两种形式：

### 9.1 单行紧凑型

适合 3 到 5 个筛选项。

结构：

- 左侧筛选
- 右侧操作按钮

### 9.2 多行网格型

适合 6 个以上筛选项。

结构：

- 常用筛选在前
- 日期范围单独占位
- 操作按钮固定右侧

### 9.3 推荐尺寸

- 输入框 / 选择框：`34px ~ 36px`
- 普通单选：`220px ~ 280px`
- 关键字输入：`260px ~ 320px`
- 日期区间：`320px ~ 360px`

### 9.4 交互规则

- 不自动请求
- 点击查询后请求
- 回车可查询
- 重置后清空并请求
- 分页变化可请求

### 9.5 用例管理查询栏标准布局

用例管理采用“两段式查询栏”，后续同类列表页优先参考：

第一段：筛选条件行。

- 请求方式：多选，宽度按栅格 `lg=5`。
- 优先级：多选，宽度按栅格 `lg=5`。
- 自动化：单选，宽度按栅格 `lg=5`。
- 创建日期：日期区间，宽度按栅格 `lg=7`。
- 每项之间使用 `12px` gutter。
- 每个 `el-form-item` 的 `margin-bottom` 必须为 `0`。

第二段：关键词 + 操作按钮行。

- 左侧关键词输入：宽度 `280px` 到 `320px`。
- 右侧按钮组：查询、重置、新增。
- 按钮高度：`34px`。
- 普通按钮最小宽度：`76px`。
- 新增按钮使用主视觉，允许品牌渐变。
- 按钮组必须右对齐，不允许散落在查询项中间。

查询区面板：

- 内边距：`14px`。
- 内部行间距：`12px`。
- 使用玻璃面板背景，不能是纯白或纯黑实心块。

### 9.6 查询栏禁止项

- 禁止筛选项变化立即请求，除非是左侧分类树点击。
- 禁止查询按钮和重置按钮换行到下一行。
- 禁止日期范围被拉伸到整行。
- 禁止把“新增”按钮放到表格内部右上角造成视觉漂移。
- 禁止查询面板高度过高导致首屏列表被挤压。

## 10. 列表与表格规范

### 10.1 推荐列宽

- 编号：`140px ~ 170px`
- 名称：`180px ~ 240px`
- 方法 / 状态：`80px ~ 120px`
- 长文本：`220px ~ 320px`
- 日期：`140px ~ 160px`
- 创建人：`90px ~ 120px`
- 操作：`120px ~ 150px`

### 10.2 规则

- 编号和名称分列展示
- 长文本统一用 `OverflowText`
- 操作列固定且紧凑
- hover 高亮与主题一致
- 空状态需要统一样式

### 10.3 用例管理字段排列基准

用例管理表格是后续复杂列表页的字段分布参考。字段顺序和宽度如下：

| 字段 | 宽度 | 固定 | 对齐 | 说明 |
| --- | --- | --- | --- | --- |
| 勾选框 | `42px` | 左固定 | 居中 | 批量操作入口 |
| 编号 | `162px` | 左固定 | 左对齐 | 使用等宽字体，展示 `APICASE-YYYYMMDD01` |
| 用例名称 | `min-width: 190px` | 左固定 | 左对齐 | 主识别字段，可点击编辑 |
| Method | `84px` | 不固定 | 居中 | 标签居中，表头排序不换行 |
| URL | `min-width: 260px` | 不固定 | 左对齐 | 主长文本列，使用等宽字体 |
| 描述 | `min-width: 220px` | 不固定 | 左对齐 | 次长文本列 |
| 优先级 | `86px` | 不固定 | 居中 | 使用圆点 + P0/P1/P2/P3 |
| 自动化 | `118px` | 不固定 | 居中 | 使用 `el-switch` |
| 更新时间 | `150px` | 不固定 | 左对齐 | 固定日期时间宽度 |
| 创建人 | `96px` | 不固定 | 左对齐 | 内容过长省略 |
| 操作 | `136px` | 右固定 | 居中 | 编辑、复制、删除一行展示 |

功能用例差异：

| 字段 | 宽度 | 说明 |
| --- | --- | --- |
| 前置条件 | `min-width: 240px` | 替代 URL |
| 描述 | `min-width: 190px` | 次长文本 |
| 步骤数 | `84px` | 居中展示 |

字段排列原则：

- 识别字段在左侧：勾选框、编号、名称。
- 类型字段靠近名称：Method / 步骤数。
- 长文本放中间：URL、描述、前置条件。
- 状态字段靠右：优先级、自动化。
- 审计字段更靠右：更新时间、创建人。
- 操作永远最右固定。

### 10.4 表格透明度强约束

表格不能使用 Element Plus 默认实心背景，否则页面背景和面板流动感会被盖住。

深色主题表格必须接近以下配置：

```css
.table-panel :deep(.el-table) {
  --el-table-bg-color: transparent;
  --el-table-tr-bg-color: rgba(8, 18, 32, 0.34);
  --el-table-header-bg-color: rgba(15, 31, 52, 0.46);
  --el-table-expanded-cell-bg-color: rgba(8, 18, 32, 0.42);
  background:
    linear-gradient(rgba(56, 189, 248, 0.035) 1px, transparent 1px),
    linear-gradient(90deg, rgba(56, 189, 248, 0.03) 1px, transparent 1px),
    rgba(8, 18, 32, 0.32);
  background-size: 28px 28px, 28px 28px, auto;
}
```

浅色主题表格必须接近以下配置：

```css
html:not(.dark) .table-panel :deep(.el-table) {
  --el-table-tr-bg-color: rgba(255, 255, 255, 0.54);
  --el-table-header-bg-color: rgba(240, 247, 255, 0.68);
  --el-table-expanded-cell-bg-color: rgba(255, 255, 255, 0.64);
  background:
    linear-gradient(rgba(22, 119, 255, 0.035) 1px, transparent 1px),
    linear-gradient(90deg, rgba(22, 119, 255, 0.03) 1px, transparent 1px),
    rgba(255, 255, 255, 0.44);
  background-size: 28px 28px, 28px 28px, auto;
}
```

行背景规则：

- 表头高度：`44px`。
- 数据行高度：`48px`。
- 单元格左右内边距：`10px`。
- 偶数行深色：`rgba(15, 31, 52, 0.28)`。
- 偶数行浅色：`rgba(245, 250, 255, 0.5)`。
- hover：统一使用 `var(--color-primary-soft)`。
- 左右固定列 hover 背景必须同步覆盖，避免出现半行高亮。

### 10.5 表格内部流动感

表格面板必须有自己的背景和扫描层：

- 面板背景网格：`32px * 32px`。
- 表格背景网格：`28px * 28px`。
- 扫描动画：`12s linear infinite`。
- 扫描层透明度：`0.8` 左右。
- 扫描层使用 `pointer-events: none`。

参考结构：

```css
.table-panel {
  position: relative;
  display: flex;
  flex: 1;
  min-height: 0;
  flex-direction: column;
  background:
    linear-gradient(rgba(56, 189, 248, 0.05) 1px, transparent 1px),
    linear-gradient(90deg, rgba(56, 189, 248, 0.045) 1px, transparent 1px);
  background-size: 32px 32px;
}

.table-panel::before {
  position: absolute;
  inset: 0;
  pointer-events: none;
  background:
    linear-gradient(110deg, transparent 0 36%, rgba(56, 189, 248, 0.12) 50%, transparent 66%),
    radial-gradient(circle at 88% 16%, rgba(34, 211, 166, 0.14), transparent 26%);
  opacity: 0.8;
  content: "";
  animation: table-scan 12s linear infinite;
}
```

### 10.6 表格操作区

- 表格行内操作按钮必须一行展示。
- 操作按钮字号：`12px`。
- 操作按钮左右 padding：`0 3px`。
- 按钮间距：`4px`。
- 操作列宽不足时，优先增加操作列宽，不允许换行。

### 10.7 编号规则

- 接口用例：`APICASE-YYYYMMDD01`
- 功能用例：`FUNCCASE-YYYYMMDD01`

## 10.8 不允许遗漏清单

模型调整页面时，必须逐项检查：

- 页面背景是否有细网格。
- 页面背景是否有轻扫描线。
- 页面背景是否有极淡节点漂移。
- 面板内部是否仍能看到纹理。
- 表格是否透明，而不是默认实心底色。
- 表格是否有内部网格和扫描层。
- 查询栏是否分布合理，按钮是否右对齐。
- 日期范围是否固定宽度。
- 表格字段是否按“识别 -> 类型 -> 长文本 -> 状态 -> 审计 -> 操作”排列。
- 编号和名称是否分列。
- 操作列是否一行展示。
- hover 高亮是否覆盖固定列。

## 11. 弹窗规范

### 11.1 分层

- 基础信息
- 核心配置
- 高级配置

### 11.2 推荐字段分布

基础信息：

- 名称
- 说明
- 前置条件
- 优先级

核心配置：

- 接口请求配置
- 功能步骤配置
- 主业务字段

高级配置：

- 自动化
- 脚本路径
- 入口函数
- 参数
- 标签
- 失败策略
- 超时配置

### 11.3 字段数量判断

- 1 到 6 个字段：单列或双列
- 7 到 12 个字段：双列 + 分区
- 超过 12 个字段：折叠或标签页

## 12. 主题适配

必须验证：

- 输入框
- 下拉框
- 日期框
- tooltip
- table hover
- dialog
- 选中态

要求：

- 不写死颜色
- 所有颜色优先走变量
- 浮层必须兼容深浅色
- 不允许主题切换后出现高亮污染

## 13. Vibecoding 顺序

建议按以下顺序生成页面：

1. 页面骨架
2. 查询区
3. 列表区
4. 弹窗区
5. 背景与动效

没有的模块直接删，不要硬造。

## 15. 验收标准

完成改造后，页面应满足：

- 文字清晰
- 布局均匀
- 查询栏整齐
- 表格可扫读
- 弹窗分区明确
- 深浅色主题一致
- tooltip 正常
- 没有样式污染
